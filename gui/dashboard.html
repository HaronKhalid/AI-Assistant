How to Connect Dashboard to ARIA Backend
1
Place dashboard.html in your ARIA project
Copy the dashboard file into your ~/aria/gui/ folder alongside main.py
mkdir -p ~/aria/gui cp ~/Downloads/aria_dashboard.html ~/aria/gui/dashboard.html
2
Install the FastAPI bridge server
This creates a local WebSocket server that connects the dashboard to Python
source ~/aria/venv/bin/activate pip install fastapi uvicorn websockets python-multipart
3
Create the bridge server (api_server.py)
Save this as ~/aria/api_server.py — it bridges dashboard ↔ ARIA Python
from fastapi import FastAPI, WebSocket from fastapi.staticfiles import StaticFiles import uvicorn, asyncio, json app = FastAPI() app.mount("/", StaticFiles(directory="gui", html=True), name="gui") clients = set() @app.websocket("/ws") async def ws(ws: WebSocket): await ws.accept() clients.add(ws) try: while True: data = await ws.receive_text() # Forward to ARIA main loop print(f"[Dashboard] {data}") await ws.send_text(json.dumps({"type":"ack","msg":"received"})) finally: clients.remove(ws) if __name__ == "__main__": uvicorn.run(app, host="127.0.0.1", port=8765)
4
Run the server and open dashboard
Run both ARIA and the web server. Dashboard opens in any browser.
# Terminal 1 — start Ollama ollama serve # Terminal 2 — start ARIA cd ~/aria && source venv/bin/activate python main.py --no-wake # Terminal 3 — start dashboard server python api_server.py # Then open in browser: xdg-open http://localhost:8765
5
Or use as a standalone file (current mode)
The dashboard works right now as a standalone HTML file. Quick commands simulate real responses. To connect live data, add the WebSocket client code to this file's JavaScript and point it to ws://localhost:8765/ws
xdg-open ~/aria/gui/dashboard.html # Opens in Firefox/Chrome with full UI # All animations and interactions work offline
